# Aspend Auto-Detection Workflow

This document explains how Aspends detects transactions automatically across different sources and ensures 100% accuracy.

## 1. Detection Sources & Native Listeners
The app monitors three primary sources on Android:

### A. SMS Receiver (`SmsReceiver.kt`)
- Listens for `android.provider.Telephony.SMS_RECEIVED`.
- Extracts: Sender, Body, and Timestamp.
- Status: Active in background.

### B. Notification Listener (`TransactionDetectionService.kt`)
- Listens for status bar notifications from all apps.
- Filters: Ignores own app notifications.
- Extracts: Title, Text, BigText, and Package Name.
- Status: Requires "Notification Access" permission.

### C. Accessibility Service (`TransactionAccessibilityService.kt`)
- Listens for window changes in payment apps (GPay, PhonePe, Paytm, etc.).
- Mode: Real-time screen scraping of UI nodes.
- Extracts: Direct on-screen text and package name.
- Status: Requires "Accessibility" permission.

```mermaid
    G --> H[TransactionParser.parse]
    H --> I{Match Found?}
    I -- Yes --> J[Deduplication Check: 10min Window/Ref ID]
    J -- New --> K[Save to Hive & Update Balance]
    J -- Duplicate --> L[Ignore/Log]
    I -- No --> M[Record in Detection History: 'skipped']
```

## 3. Advanced Parsing Logic (`TransactionParser.dart`)

The parser uses a multi-stage approach for "Advance Level" accuracy:

1.  **Validity Filter**: Immediately discards OTPs, failed/declined messages, and non-financial alerts.
2.  **Amount Extraction**: Priority-based regex for currency markers (Rs. ₹, INR) and amounts.
3.  **Account Validation**:
    - Strictly identifies 3-6 digit account numbers.
    - **Year Filter**: Prevents years like "2024" from being identified as accounts unless explicitly marked.
    - **Amount Conflict**: Ensures the amount itself isn't grabbed as the account number.
4.  **Merchant Extraction**:
    - **UPI Priority**: Extracts UPI handles (e.g., merchant@okaxis) as the highest confidence source.
    - **Directional Parsing**: Uses keywords like "to", "at", "towards", "on" to pinpoint merchant names.
    - **Location Cleaning**: Strips city names (Bangalore, Mumbai) and generic noise (Mobile, Card).
5.  **Balance Conflict Resolution**: If a balance suffix matches the account number in a small window, it re-verifies the account marker before the balance keyword to prevent false positives.

## 5. Robustness Features
- **Native Queuing**: If the app is killed, native services save events to a local queue so they are processed immediately upon next launch.
- **Double-Check Balance**: The system automatically avoids picking up the "Available Balance" as the "Amount Debited" by checking for specific balance keywords first.
- **Deduplication**: If an SMS and a Notification arrive for the same ₹500 transaction within 10 minutes, only one is added to the database.

---

## 6. Real-time Success Capture

Aspend features a specialized detection mode for live transactions:

- **Success Detection**: Accessibility service monitors for keywords like "Payment Successful", "Transaction Successful", and "Paid" within target payment apps.
- **Auto-Screenshot**: Upon detection, the app automatically triggers a native screenshot (API 30+).
- **Persistent Storage**: The screenshot is saved to the app's secure data directory and linked directly to the newly created transaction.
- **Visual Verification**: Users can view the original payment success screen within Aspend to verify transaction details.

> [!NOTE]
> Screenshots are only captured on screens explicitly identified as "Success" screens to protect user privacy during PIN entry or navigation.
